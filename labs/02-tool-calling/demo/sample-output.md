# نمونه خروجی — Tool Calling Agent

این فایل نشان می‌دهد Agent دوم چگونه با کمک مدل زبانی، ابزار مناسب را انتخاب می‌کند و خروجی نهایی فارسی می‌سازد.

---

## مثال 1 — سختی یادگیری

### ورودی کاربر

```text
سختی یادگیری LangGraph چقدر است؟
```

### ابزار انتخاب‌شده توسط Agent

```text
learning_difficulty
```

### خروجی ابزار

```text
سطح سختی یادگیری LangGraph حدود 8 از 10 است.
```

### پاسخ نهایی Agent

```text
یادگیری LangGraph نسبتاً پیشرفته است و حدود 8 از 10 سختی دارد، چون برای استفاده درست از آن باید با مفاهیمی مثل state، graph، workflow و orchestration آشنا باشید.
```

---

## مثال 2 — خلاصه‌سازی موضوع

### ورودی کاربر

```text
یک خلاصه کوتاه درباره RAG بده
```

### ابزار انتخاب‌شده توسط Agent

```text
topic_summary
```

### خروجی ابزار

```text
RAG یکی از مفاهیم یا ابزارهای مهم در مسیر ساخت AI Agent است و باید با مثال عملی یاد گرفته شود.
```

### پاسخ نهایی Agent

```text
RAG روشی است که به Agent کمک می‌کند قبل از پاسخ دادن، از منابع بیرونی مثل فایل، مستندات یا پایگاه دانش اطلاعات مرتبط را پیدا کند و بعد پاسخ دقیق‌تری بسازد.
```

---

## مثال 3 — تشخیص نوع موضوع

### ورودی کاربر

```text
نوع CrewAI چیست؟
```

### ابزار انتخاب‌شده توسط Agent

```text
topic_type
```

### خروجی ابزار

```text
Framework
```

### پاسخ نهایی Agent

```text
CrewAI یک Framework برای ساخت سیستم‌های Multi-Agent است؛ یعنی می‌توان با آن چند Agent با نقش‌های مختلف ساخت که با هم روی یک هدف کار کنند.
```

---

## نکته

این Agent یک نمونه ساده اما واقعی از Tool Calling است.
در این نسخه، مدل زبانی تصمیم می‌گیرد کدام ابزار باید اجرا شود.
